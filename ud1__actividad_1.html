UF2217. UD1. Actividad 1. Caso práctico: DOCUMENTOS XML BIEN FORMADOS


Enunciado (Descripción)

Se facilitan 7 sencillos documentos XML que no están bien formados, es decir, no cumplen alguna de las normas básicas a nivel de sintaxis. Se pide la corrección de los documentos para obtener documentos XML bien formados.

Podrás descargar y acceder a estos documentos en la carpeta de archivos adjuntos:

  • cantantesMalFormado.xml
  • capituloMalFormado.xml
  • circuitosMalFormado.xml
  • festivalMalFormado.xml
  • formatosMalFormado.xml
  • librosMalFormado.xml
  • programasMalFormado.xml 

Archivo adjunto: UF2217_UD1_Actividad 1_Caso práctico_AA.rar


Preguntas / Actividades a realizar 

Partiendo de los documentos de apoyo, detectar los incumplimientos de la sintaxis que debe tener un documento xml y corregirlos. Es posible que para corregir alguno de los errores haya que añadir un atributo o etiqueta; en este caso, añadir la etiqueta o el atributo con un nombre que tenga relación con el ejemplo en que nos encontramos.

Una vez corregidos, hacer uso de “XML Copy Editor” para comprobar que son documentos bien formados.

Para la evaluación de la actividad será preciso insertar capturas de pantalla de cada una de las comprobaciones que se realizan de los 7 documentos con la herramienta de XML Copy Editor.

UD1 - Actividad 1 - Resolución

Pulsa aquí para visualizar: UD1 - Actividad 1 - Resolución

cantantesMalFormado

El documento XML proporcionado presenta varios errores de sintaxis:

  • Etiquetas de cierre incorrectas: las etiquetas de cierre deben comenzar con "</" seguido del nombre del elemento, en el documento proporcionado, algunas etiquetas de cierre están incompletas o mal escritas.
  • Uso incorrecto de las etiquetas de apertura y cierre: las etiquetas de apertura y cierre deben coincidir y ser coherentes entre sí.
  • Atributos sin comillas: los valores de los atributos deben estar entre comillas dobles (" ").

capituloMalFormado

El documento XML proporcionado tiene varios errores de sintaxis:

  • Uso incorrecto de etiquetas en minúsculas: en XML, las etiquetas son sensibles a mayúsculas y minúsculas, por lo que , , y <http://> deben estar en minúsculas para que coincidan con sus etiquetas de apertura correspondientes.
  • Etiqueta <año> sin comillas: el valor "1645" dentro de la etiqueta <año> debe estar entre comillas dobles para cumplir con la sintaxis de XML.
  • Etiqueta <http://> incorrecta: la etiqueta <http://> no sigue la convención de nombre de etiqueta válida en XML y tampoco tiene contenido adecuado. Parece que se intenta proporcionar una URL, pero esto debe hacerse de una manera diferente en XML.
  • Falta de etiqueta de cierre para : la etiqueta debe tener una etiqueta de cierre correspondiente.

circuitosMalFormado

El documento XML proporcionado tiene varios errores de sintaxis:

  • Espacios en el nombre de la etiqueta raíz: el nombre de la etiqueta raíz contiene espacios en blanco, los nombres de las etiquetas XML no pueden contener espacios en blanco, deben ser palabras únicas y sin espacios.
  • Uso incorrecto de etiquetas de apertura y cierre: la etiqueta de apertura no tiene una etiqueta de cierre correspondiente, además, la etiqueta de apertura incluye un atributo id que no está correctamente formateado.
  • Formato incorrecto de la distancia: el valor "3,4 Km" dentro de la etiqueta está utilizando una coma como separador decimal, lo cual no es válido en XML, se debe utilizar un punto como separador decimal en números.
  • Redundancia en la estructura de datos: la etiqueta se repite tanto como etiqueta contenedora como etiqueta de contenido, esto puede causar confusión y no es necesario en la estructura del documento.

festivalMalFormado

En este caso y previo al análisis XML, el primero error es claramente de elección para ilustrar este ejemplo, ya que la candidatura de Suecia a Eurovisión en el año 2005 tiene el dudoso honor de ser la canción con peor puntuación en una final de Eurovisión en el últimos 30 años del país nórdico.

Dicho lo cual, el documento XML proporcionado tiene varios errores de sintaxis:

  • Espacios en los nombres de las etiquetas: en XML, los nombres de las etiquetas no pueden contener espacios.
  • Atributos mal formados: los atributos en XML deben estar dentro de una etiqueta de inicio, no pueden ser una etiqueta por sí mismos.
  • Nombres de etiquetas numéricas: en XML, los nombres de las etiquetas no pueden comenzar con un número.

formatosMalFormado

 

El documento XML proporcionado tiene un problema principal que impide que sea bien formado:

  • Anidamiento incorrecto de las etiquetas: en XML, las etiquetas deben estar correctamente anidadas. Esto significa que las etiquetas que se abren primero deben cerrarse en último lugar, en el documento, la etiqueta  se abre después de la etiqueta  pero ésta se cierra antes.

librosMalFormado

Sin duda alguna el autor o autora de estos ejercicios debería reflexionar sobre la idoneidad de los ejemplos. Si una persona de origen musulmán debe realizar esta actividad, ¿cómo se sentiría?.

Soy fiel defensor y ultraortodoxo de la libertad de expresión, además personalmente me parece una obra extraordinariamente valiente. Y desde luego, no seré yo quien enarbole la bandera pro islámica frente a la censura y la sinrazón propias del islamismo radical. Ahora bien, entre los cientos de miles de millones de billones de libros que hay, ¿es necesario escoger la obra "Versos satánicos" para ilustrar una actividad de lenguaje XML?.

Dicho lo cual, el documento XML proporcionado presenta varios errores de sintaxis:

  • No hay un elemento raíz único: un documento XML debe tener un solo elemento raíz que contenga todos los demás elementos.
  • Etiqueta mal formada: la etiqueta de cierre está fuera de lugar y no tiene una etiqueta de apertura correspondiente.

programasMalFormado

El documento XML proporcionado presenta errores de sintaxis:

  • La etiqueta <programa> tiene dos atributos canal, lo que no es válido en XML. Los atributos de un elemento XML deben ser únicos.
  • Hay un espacio adicional al final del valor del atributo canal en el primer elemento <programa>, lo que puede causar inconsistencias en la interpretación de los datos.
