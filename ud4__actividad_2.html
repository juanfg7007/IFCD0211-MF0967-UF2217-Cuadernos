UF2217. UD4. Actividad 2. Caso práctico: VALIDACIONES DE ESQUEMA.


Enunciado (Descripción)

Se proponen a continuación una serie de documentos XML no válidos. Hay que hacer que sean válidos corrigiendo la DTD.


Cd.xml

<!ELEMENT cd EMPTY>

<!ATTLIST cd cancion NMTOKEN #REQUIRED>

]>


Pintores.xml

<!ELEMENT inventores>

<!ELEMENT pintor EMPTY>

<!ATTLIST pintor cuadro CDATA #REQUIRED>

<!ATTLIST pintor nombre ID #REQUIRED>

]>


agendaTrabajo.xml

<!ELEMENT agendaTrabajo (tarea)>

<!ELEMENT tarea EMPTY>

<!ATTLIST tarea fechaInicio CDATA #REQUIRED>

<!ATTLIST tarea descripcion CDATA #REQUIRED>

<!ATTLIST tarea fechaFin CDATA #REQUIRED>

]>

Pulsa aquí para visualizar: UD4 - Actividad 2 - Resolución

Cd.xml

El documento XML proporcionado define un elemento cd con un atributo cancion. Sin embargo, en la definición de tipo de documento (DTD), se especifica que el elemento cd es EMPTY, lo que significa que no puede tener contenido ni atributos. Esto es lo que hace que el documento XML sea inválido.

Pintores.xml

El documento XML proporcionado define un elemento pintores que contiene varios elementos pintor. Cada elemento pintor tiene dos atributos: nombre y cuadro. Sin embargo, en la definición de tipo de documento (DTD), se especifica que el elemento pintor es EMPTY, lo que significa que no puede tener contenido ni atributos. Además, se ha definido un elemento inventores que no se utiliza en el documento XML. Estos son los problemas que hacen que el documento XML sea inválido.

AgendaTrabajo.xml

El documento XML proporcionado define un elemento agendaTrabajo que contiene varios elementos tarea. Cada elemento tarea tiene tres atributos: fechaInicio, descripcion y fechaFin, y también contiene texto. Sin embargo, en la definición de tipo de documento (DTD), se especifica que el elemento tarea es EMPTY, lo que significa que no puede tener contenido ni atributos. Esto es lo que hace que el documento XML sea inválido.
